# System Patterns - Плодовка

## Архитектура системы

### Общая схема
```
Пользователь → Браузер → index.html (статический сайт)
                                    ↓
                            Google Таблица (CSV)
                                    ↓
                            PapaParse (парсинг)
                                    ↓
                            Рендеринг товаров
```

### Компоненты системы

1. **Frontend (index.html)**
   - Единый HTML файл со встроенным CSS и JavaScript
   - Нет сборки, нет фреймворков
   - Статический сайт, деплоится на GitHub Pages

2. **Data Layer**
   - Источник: Google Таблица (публичная, экспортируется как CSV)
   - Парсинг: PapaParse (библиотека через CDN)
   - Структура: Наименование, Группа, Цена, На_складе, Фото1, Фото2, Видео

3. **UI Layer**
   - Tailwind CSS (utility-first CSS framework через CDN)
   - Кастомные стили в <style> блоке
   - Адаптивная сетка (grid + flexbox)

4. **State Management**
   - Vanilla JavaScript объекты
   - productsData - массив товаров
   - cart - объект корзины
   - currentCategory, searchQuery - фильтры

## Ключевые паттерны проектирования

### 1. Паттерн "Единый источник данных"
- Все товары загружаются из Google Таблицы
- Нет дублирования данных в коде
- Обновление ассортимента через редактирование таблицы

### 2. Паттерн "Рендеринг на клиенте"
- Данные парсятся в браузере
- Динамическое создание DOM элементов
- Skeleton loading во время загрузки

### 3. Паттерн "Модальные окна"
- Детали товара: productModal
- Корзина: cartModal
- Лайтбокс для фото: lightboxModal
- Toast уведомления

### 4. Паттерн "Observer"
- Intersection Observer для анимаций при скролле
- Анимация появления карточек товаров
- Анимация счётчиков статистики

### 5. Паттерн "Event Delegation"
- Обработчики событий на родительских элементах
- Остановка всплытия для кнопок в карточках

### 6. Паттерн "Local Storage"
- Сохранение выбранной темы (светлая/тёмная)
- Восстановление при следующем посещении

## Структура кода

### HTML
```
<head>
  - Meta теги (SEO, Open Graph, Twitter Card)
  - Tailwind CSS (CDN)
  - PapaParse (CDN)
  - Google Fonts
  - Кастомные стили
</head>
<body>
  - Header (навигация, поиск, корзина)
  - Main (фильтры, каталог товаров)
  - FAQ секция
  - Статистика
  - Преимущества
  - Footer
  - Модальные окна
  - JavaScript
</body>
```

### JavaScript модули (внутри <script>)
1. **Конфигурация** - URL таблицы, профиль Авито
2. **Состояние** - глобальные переменные
3. **Утилиты** - extractUrls, getYouTubeEmbedUrl, parseName
4. **Загрузка данных** - loadProductsFromSheet
5. **Рендеринг** - renderCategoryFilters, renderProducts
6. **Модалки** - openProductModal, showModalMedia, toggleCartModal
7. **Корзина** - addToCart, updateCartQuantity, updateCartUI
8. **Заказ** - copyAndRedirectToAvito
9. **UI** - showToast, toggleTheme, resetFilters
10. **Анимации** - animateCounters, Intersection Observers

## Критические пути реализации

### Загрузка товаров
1. window.onload → loadProductsFromSheet()
2. Показ skeleton loading (6 карточек)
3. Papa.parse() → загрузка CSV
4. Обработка данных (фильтрация, парсинг)
5. renderCategoryFilters() + renderProducts()
6. Intersection Observer начинает отслеживать карточки

### Добавление в корзину
1. Клик на кнопку "+" → addToCart(id)
2. Проверка наличия на складе
3. Увеличение количества или создание новой записи
4. updateCartUI() - обновление счётчика и списка
5. Анимация бейджа корзины
6. Toast уведомление

### Оформление заказа
1. Клик "Оформить заказ" → copyAndRedirectToAvito()
2. Проверка что корзина не пуста
3. Формирование текста заказа
4. Копирование в буфер обмена
5. Toast уведомление
6. Закрытие модалки корзины
7. Открытие Авито в новой вкладке

## Дизайн-система

### Цвета
- Основной: emerald-800 (#047857)
- Акцентный: yellow-200 (для логотипа)
- Фон: slate-50 (светлая), slate-900 (тёмная)
- Текст: slate-800/900 (основной), slate-400/500 (вторичный)

### Типографика
- Заголовки: Playfair Display (serif)
- Основной текст: Inter (sans-serif)
- Мобильные: text-sm, десктоп: text-base

### Отступы
- Контейнер: max-w-7xl, px-4
- Карточки: p-5/p-6
- Секции: py-8/py-10/py-16

### Анимации
- fadeInUp: появление элементов
- shimmer: skeleton loading
- cartBounce: добавление в корзину
- modalFadeIn: открытие модалок
- countPop: анимация счётчиков

## Обработка ошибок

### Текущее состояние
- Обработка ошибок парсинга CSV (alert + сообщение)
- Проверка наличия товара перед добавлением в корзину
- Fallback для товаров без фото

### Что можно улучшить
- Офлайн-режим (Service Worker частично реализован)
- Кэширование данных в localStorage
- Повторные попытки загрузки при ошибке сети
- Понтные сообщения об ошибках вместо alert()

## Безопасность

- Нет чувствительных данных в коде
- Все внешние ссылки с rel="noopener noreferrer"
- Нет инъекций (данные экранируются при рендеринге)
- HTTPS на GitHub Pages
