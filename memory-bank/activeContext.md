## Активные решения и соображения

1. **Платформа заказа:** Только Авито (без собственной оплаты)
   - Причина: проще для старта, доверие покупателей
   - Альтернатива: собственный сайт с оплатой (в будущем)

2. **Источник данных:** Google Таблица + PapaParse
   - Причина: легко обновлять ассортимент без изменения кода
   - Риск: зависимость от внешнего сервиса

3. **Дизайн:** Tailwind CSS
   - Причина: быстро разработать, легко поддерживать
   - Все стили через utility classes

4. **Без фреймворков:** Vanilla JavaScript
   - Причина: проще деплоить на GitHub Pages
   - Нет сборки, нет зависимостей

## Важные паттерны и предпочтения

- **Код:** чистый, хорошо прокомментированный
- **Структура:** один файл index.html (всё включено)
- **Данные:** выгрузка CSV из Google Таблицы
- **Стили:** Tailwind CSS через CDN
- **Иконки:** inline SVG (не библиотека)
- **Шрифты:** Google Fonts (Inter + Playfair Display)

## Learnings и инсайты проекта

1. **Пользователи:** основная аудитория - дачники 25-65 лет, важна простота использования
2. **Конверсия:** минимальное количество шагов для заказа = выше конверсия
3. **Мобильность:** 70%+ трафика с мобильных, критична адаптивность
4. **Доверие:** FAQ с гарантиями и контактами повышает уверенность
5. **Ассортимент:** легко управлять через Google Таблицу, но нужна валидация данных

## Блокеры и проблемы

- Нет блокеров на текущий момент
- Зависимость от доступности Google Таблицы (если недоступна - сайт не работает)
- Нет обработки ошибок при недоступности CSV (можно улучшить)
